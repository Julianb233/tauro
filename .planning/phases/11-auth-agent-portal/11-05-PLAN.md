# Plan 11-05: Agent Manager + Tour Calendar

**Phase:** 11 -- Auth & Agent Portal
**Plan ID:** 11-05
**Status:** PENDING
**Depends On:** 11-02 (dashboard shell)

## Goal

Build the Agent Manager page (admin-only, list and edit agents) and the Tour Calendar page (view booked showings from leads with type=showing in day and week views).

## Requirements Addressed

- **AUTH-06**: Agent Manager (admin only) -- add/edit agents, view performance, manage roles
- **AUTH-07**: Tour Calendar -- view booked showings by date, agent, property (day/week)

## Context

- AgentRow type: id, slug, first_name, last_name, full_name, title, email, phone, photo, bio, specialties, neighborhoods, stats, awards, etc.
- Existing `GET /api/agents` returns all agents (no pagination yet, small dataset)
- LeadRow with type="showing" contains metadata with preferredDate/preferredTime fields
- getUserProfile() and getUserRole() from auth.ts to check admin access
- ProfileRow has role field and agent_id linking to agents table
- No external calendar library -- build a simple custom calendar component (showing data is limited, no need for full-calendar)

  <name>Task 1: Agent Manager page (admin only)</name>
  <files>
    src/app/dashboard/agents/page.tsx

  <action>
  1. Create `src/app/api/agents/[id]/route.ts`:
     - GET handler: return single agent by ID with their active listings count
       - Query: `*, properties!listing_agent_id(id)` then count properties
       - 404 if not found
     - PATCH handler: accepts partial agent fields
       - Updatable fields: first_name, last_name, title, email, phone, photo, bio, short_bio, specialties, neighborhoods, video_intro_url, social, languages, license_number
       - Auto-update full_name from first_name + last_name if either changes
       - Set updated_at to current timestamp
       - Return updated agent row
     - DELETE handler: delete agent by ID (only if no linked properties or leads)
       - Check for FK references first, return 409 Conflict if agent has active properties or leads assigned
       - Return 204 on success

  2. Create `src/app/dashboard/agents/page.tsx`:
     - Server component for initial data fetch, with client interactivity wrapper
     - FIRST: check user role with getUserRole(). If not "admin", show "Access Denied. Admin only." message with link back to /dashboard. Do NOT redirect (avoids loops).
     - Page heading: "Agent Manager" with "Add Agent" button (if implementing add -- for MVP, just edit existing)
     - Agent list as cards or table rows, each showing:
       - Photo (circular, 48px) or initials avatar
       - Full name (bold)
       - Title (e.g., "Senior Agent")
       - Email and phone
       - Active listings count (query from properties)
       - Number of assigned leads
       - Edit button (pencil icon)
     - Edit functionality: clicking Edit opens an inline edit form or modal with editable fields:
       - First Name, Last Name, Title, Email, Phone
       - Bio (textarea)
       - Specialties (tag input, same pattern as property features)
       - Neighborhoods (tag input)
       - Photo (ImageUpload with bucket="agent-photos")
     - Save calls PATCH /api/agents/[id], refresh list
     - Cancel reverts to display mode
     - Style: dark cards matching Tauro brand, gold accent on edit buttons
  </action>
  <verify>
    Run `npm run build`. As admin user, navigate to /dashboard/agents. Verify agent list shows all agents with stats. Click Edit, change title, save. Verify change persists. As non-admin, verify "Access Denied" message appears.
  </verify>
  <done>
    - Agent Manager page lists all agents with photos, stats, and edit buttons
    - Admin-only access enforced (non-admins see access denied)
    - Edit form updates agent data via PATCH endpoint
    - Agent API endpoints (GET/PATCH/DELETE) work correctly
  </done>

  1. Create `src/components/dashboard/calendar-view.tsx`:
     - "use client" component
     - Props: `events: ShowingEvent[]`, `view: "day" | "week"`, `currentDate: Date`, `onDateChange: (date: Date) => void`, `onViewChange: (view: "day" | "week") => void`

     Define ShowingEvent type:
     ```
     { id: string; leadName: string; email: string; phone: string; propertyAddress: string; propertyId?: string; agentName?: string; date: string; time: string; status: string }
     ```

     **Week view (default):**
     - Show 7-day grid (Mon-Sun) for the selected week
     - Each day column shows the day name and date number
     - Events rendered as small cards within their day column:
       - Time (bold)
       - Visitor name
       - Property address (truncated)
       - Status badge (colored dot)
     - Click event card to expand showing detail (inline expand or tooltip)
     - Today's column has subtle gold left border

     **Day view:**
     - Time slots from 8am to 8pm (hourly rows)
     - Events placed at their time slot
     - Each event shows: time, visitor name, property, agent, phone
     - More spacious layout than week view

     **Navigation:**
     - Left/right arrows to go to previous/next week or day
     - "Today" button to jump to current date
     - View toggle: "Day" | "Week" buttons
     - Current date range displayed as text (e.g., "Mar 18 - Mar 24, 2026" for week, "March 18, 2026" for day)

     **Styling:**
     - Dark background grid
     - Event cards: dark card with left border color based on status
     - Gold accent for today, current time indicator
     - Responsive: on mobile, default to day view, week view scrolls horizontally

  2. Create `src/app/dashboard/calendar/page.tsx`:
     - "use client" component
     - Page heading: "Tour Calendar"
     - Fetch showing leads from database: query leads where type = 'showing'
       - Use GET /api/leads?type=showing or direct Supabase query
       - Extract date/time from metadata.preferredDate and metadata.preferredTime
     - Transform leads into ShowingEvent[] format:
       - leadName = first_name + " " + last_name
       - date = metadata.preferredDate (or created_at date if no preferred date)
       - time = metadata.preferredTime (or "TBD" if not specified)
       - propertyAddress = metadata.propertyAddress or "Not specified"
       - agentName = fetch from agent_id relationship
       - status = lead status
     - Pass events to CalendarView component
     - State: currentDate, view ("day" | "week")
     - Filter bar (optional, above calendar):
       - Agent filter dropdown (filter showings by assigned agent)
       - Status filter (show all, or only active/non-closed)
     - Empty state: "No showings scheduled" with link to leads page
  </action>
