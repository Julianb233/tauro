# Plan 11-02: Dashboard Shell

**Phase:** 11 -- Auth & Agent Portal
**Plan ID:** 11-02
**Status:** PENDING
**Depends On:** 11-01 (auth setup, middleware, login page)

## Goal

Build the authenticated dashboard layout shell: sidebar navigation, top header with user avatar/logout, breadcrumbs, and a dashboard overview page with summary stat cards. All dashboard pages will render inside this layout.

## Requirements Addressed

- **AUTH-03**: Protected /dashboard route (layout enforces auth)
- Partial **AUTH-04/05/06/07**: Navigation structure for all dashboard sections

## Context

- Auth helpers from Plan 11-01: `getUserProfile()`, `signOut()`
- ProfileRow type has `full_name`, `email`, `role`, `avatar_url`
- Tauro brand: dark theme (#1A1A1A bg), gold accents (#C9A84C), off-white text (#F5F0E8)
- Existing components: `@/components/logo`, `@/components/Breadcrumbs`
- Lucide icons available (lucide-react in package.json)
- shadcn base-nova style configured
- Existing API routes: GET /api/leads (with count), GET /api/properties (with count)

  1. Create `src/components/dashboard/sidebar.tsx`:
     - "use client" component
     - Props: `currentPath: string`, `userRole: UserRole`
     - Navigation items array:
       - Overview: icon=LayoutDashboard, href=/dashboard
       - Leads: icon=Inbox, href=/dashboard/leads
       - Properties: icon=Building2, href=/dashboard/properties
       - Agents: icon=Users, href=/dashboard/agents (only visible if role === "admin")
       - Calendar: icon=Calendar, href=/dashboard/calendar
     - Active state: gold left border + gold text when currentPath matches href
     - Default state: off-white/60 text, hover to off-white
     - Tauro logo at top of sidebar linking to "/" (public site)
     - "Agent Portal" text under logo in small gold text
     - Width: 240px on desktop, full-screen overlay on mobile
     - Mobile: hidden by default, toggled via hamburger button (state managed in layout or sidebar itself)
     - Background: #111111 or slightly darker than page bg
     - Bottom section: small "Back to site" link with ExternalLink icon

  2. Create `src/components/dashboard/header.tsx`:
     - "use client" component
     - Props: `user: { fullName: string; email: string; role: string; avatarUrl?: string | null }`, `onMenuToggle: () => void`
     - Left side: hamburger menu button (visible on mobile only, calls onMenuToggle)
     - Left side (desktop): breadcrumb area (just show current page title for now, can enhance later)
     - Right side: user avatar circle (initials if no photo, gold border), dropdown on click with:
       - User name (bold) and email (small, muted)
       - Role badge (admin=gold, agent=blue, viewer=gray)
       - Divider
       - "Log out" button that calls signOut() from @/lib/supabase/auth then router.push("/login")
     - Use a simple click-toggle dropdown (no external dropdown library needed), or use details/summary pattern
     - Background: transparent or slight dark bg, bottom border with gold/10 opacity

  3. Create `src/app/dashboard/layout.tsx`:
     - Server component
     - Call `getUserProfile()` to get current user data
     - If no profile (edge case), redirect to /login
     - Render: flex row layout
       - Sidebar on left (pass currentPath and userRole)
       - Main area: Header on top + scrollable content area below
     - Pass user data to Header component
     - The `children` prop renders in the content area with padding
     - Content area has max-width (e.g., 1400px) and is centered
     - For mobile sidebar toggle: wrap in a client component that manages open/close state, or use a simple approach where sidebar manages its own mobile state via a context or prop
     - Use a DashboardShell client wrapper component if needed to manage sidebar toggle state between header hamburger and sidebar
  </action>

       - Each row: name, type badge, status badge, date (relative like "2h ago")
