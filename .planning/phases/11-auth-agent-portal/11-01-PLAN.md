# Plan 11-01: Supabase Auth Setup

**Phase:** 11 -- Auth & Agent Portal
**Plan ID:** 11-01
**Status:** PENDING
**Depends On:** Phase 10 complete (Supabase client, types, queries all exist)

## Goal

Set up Supabase Auth with email/password login, role-based access (admin/agent/viewer), route protection middleware, and a login page. This is the foundation every other dashboard plan depends on.

## Requirements Addressed

- **AUTH-01**: Supabase Auth with email/password login for agents and admins
- **AUTH-02**: Role-based access control -- admin (full CRUD), agent (own leads + profile), viewer (read-only)
- **AUTH-03**: Protected /dashboard route with Next.js middleware auth guard

## Context

Existing Supabase setup:
- `src/lib/supabase/client.ts` -- browser client using `@supabase/ssr` createBrowserClient
- `src/lib/supabase/server.ts` -- server client using `@supabase/ssr` createServerClient with cookie handling
- `src/types/database.ts` -- full Database type with properties, agents, leads, etc.
- `@supabase/ssr` and `@supabase/supabase-js` already in package.json
- shadcn configured with base-nova style, Tailwind v4, lucide icons
- App uses route groups: `(site)` for public pages

No `zod` in package.json but it is used in API routes (likely installed as transitive dep or missed in package.json -- check and install if needed).

     - Add `profiles` table type to `Database["public"]["Tables"]` with columns:
       - `id` (string, matches auth.users.id)
       - `email` (string)
       - `full_name` (string)
       - `role` (Database["public"]["Enums"]["user_role"], default "viewer")
       - `agent_id` (string | null, FK to agents table -- links auth user to their agent record)
       - `avatar_url` (string | null)
       - `created_at` (string)
       - `updated_at` (string)
     - Export `ProfileRow` type alias
     - Export `UserRole` type = Database["public"]["Enums"]["user_role"]

  2. Create `src/lib/supabase/auth.ts` with these functions:
     - `signIn(email: string, password: string)` -- calls `supabase.auth.signInWithPassword()`, returns `{ user, error }`
     - `signUp(email: string, password: string, fullName: string)` -- calls `supabase.auth.signUp()` with metadata `{ full_name }`, returns `{ user, error }`
     - `signOut()` -- calls `supabase.auth.signOut()`, uses browser client
     - `getSession()` -- server-side, uses server client, returns session or null
     - `getUserProfile()` -- server-side, gets session then queries profiles table for role/agent_id, returns `ProfileRow | null`
     - `getUserRole()` -- server-side convenience, returns `UserRole | null`

     Use the existing `createClient` from `./client` for browser-side functions and from `./server` for server-side functions. Mark browser-side functions with "use client" considerations (export them from a separate section or use dynamic import pattern).

     IMPORTANT: For the server-side functions (getSession, getUserProfile, getUserRole), import createClient from "./server". For browser-side functions (signIn, signUp, signOut), create a separate browser client inline using createBrowserClient from @supabase/ssr, or import from "./client". Keep it clean -- two groups of exports clearly labeled.

  3. Create `src/lib/supabase/middleware.ts`:
     - Export `updateSession(request: NextRequest)` function that:
       a. Creates a Supabase server client with custom cookie handling for middleware (read from request, write to both request and response)
       b. Calls `supabase.auth.getUser()` to refresh the session
       c. If no user and path starts with `/dashboard`, redirect to `/login`
       d. If user is on `/login` and already authenticated, redirect to `/dashboard`
       e. Returns the response with updated cookies
     - Follow Supabase SSR middleware pattern exactly -- this is the standard approach for Next.js App Router
  </action>

    - Auth helper functions exist for signIn, signUp, signOut, getSession, getUserProfile, getUserRole
    - Middleware helper creates server client, checks auth, redirects unauthenticated users from /dashboard to /login
  </done>
</task>
