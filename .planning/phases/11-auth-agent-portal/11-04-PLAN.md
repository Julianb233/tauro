# Plan 11-04: Property Manager

**Phase:** 11 -- Auth & Agent Portal
**Plan ID:** 11-04
**Status:** PENDING
**Depends On:** 11-02 (dashboard shell)

## Goal

Build the Property Manager dashboard page where agents/admins can view all properties in a table, add new properties via a full form, edit existing properties, upload photos, toggle status, and delete with confirmation.

## Requirements Addressed

- **AUTH-05**: Property Manager -- add/edit/delete listings, upload photos, set status (active/pending/sold)

## Context

- Existing `GET /api/properties` with full filter/pagination support
- Existing `POST /api/properties` with PropertyCreateSchema validation
- PropertyRow type with all fields (address, price, beds, baths, sqft, status, images[], features, etc.)
- `ImageUpload` component at `src/components/ImageUpload.tsx` -- accepts bucket prop and onUpload callback, works with `/api/upload` endpoint
- Property statuses: "Active", "Pending", "Sold", "Open House", "New"
- Property types: "Single Family", "Condo", "Townhouse", "Multi-Family", "Land"
- AgentRow available for listing_agent_id dropdown

  <name>Task 1: Property API endpoints and table</name>
  <files>
    src/app/api/properties/[id]/route.ts
    src/app/dashboard/properties/page.tsx
    src/components/dashboard/property-table.tsx

  <action>
  1. Create `src/app/api/properties/[id]/route.ts`:
     - GET handler: return single property by ID with joined agent data
       - Select: `*, agents!listing_agent_id(id, full_name, slug)`
       - 404 if not found
     - PATCH handler: accepts partial property fields, validates with PropertyCreateSchema.partial()
       - Set updated_at to current timestamp
       - Return updated property row
     - DELETE handler: delete property by ID
       - Return 204 on success, 404 if not found
     - All handlers: proper error handling with appropriate status codes

  2. Create `src/components/dashboard/property-table.tsx`:
     - "use client" component
     - Props: `properties: PropertyRow[]`, `onEdit: (property: PropertyRow) => void`, `onDelete: (id: string) => void`, `onStatusChange: (id: string, status: string) => void`
     - HTML table with columns: Image (thumbnail), Address, Price, Beds/Baths, Status, Type, Agent, Actions
     - Image column: small thumbnail (48x48) of first image, or placeholder if no images
     - Price column: formatted as currency ($XXX,XXX)
     - Status column: colored badge + dropdown to quick-change status
       - Active=green, Pending=yellow, Sold=red, Open House=blue, New=purple
       - Clicking badge opens small dropdown with status options
       - Selecting new status calls onStatusChange
     - Actions column: Edit button (pencil icon), Delete button (trash icon, red)
     - Delete button shows confirmation: simple inline "Are you sure? Yes / No" or a small confirm dialog
     - Row hover: subtle bg change
     - Responsive: on mobile, show only Address, Price, Status, Actions columns

  3. Create `src/app/dashboard/properties/page.tsx`:
     - "use client" component
     - Page heading: "Property Manager" with count badge and "Add Property" button (top right, gold styled)
     - Fetch properties from GET /api/properties on mount
     - Render PropertyTable with properties
     - State management:
       - `showForm: boolean` -- toggles form visibility
       - `editingProperty: PropertyRow | null` -- if set, form is in edit mode
     - "Add Property" button sets showForm=true, editingProperty=null
     - Edit callback sets showForm=true, editingProperty=selected property
     - Delete callback: calls DELETE /api/properties/[id], then refreshes list
     - Status change callback: calls PATCH /api/properties/[id] with new status, refreshes list
     - After form submit (create or edit), hide form and refresh list
     - Loading state: skeleton table while fetching
     - Empty state: "No properties found. Add your first listing." with CTA button
  </action>
  <verify>
    Run `npm run build`. Navigate to /dashboard/properties. Verify table shows all properties. Test status quick-change via badge dropdown. Test DELETE with curl:
    `curl -X DELETE http://localhost:3000/api/properties/PROP_ID`
  </verify>
  <done>
    - Property table renders all properties with image thumbnails and status badges
    - Quick status change works via badge dropdown
    - Delete with confirmation removes property and refreshes table
    - GET/PATCH/DELETE API endpoints work correctly
  </done>
</task>

  1. Create `src/components/dashboard/property-form.tsx`:

     - Props: `property?: PropertyRow` (if provided, edit mode; if not, create mode), `agents: { id: string; full_name: string }[]`, `onSubmit: () => void`, `onCancel: () => void`
     - Full-page or large modal form with sections:

     **Section 1: Basic Info**
     - Address (text input, required)
     - City (text input, default "Philadelphia")
     - State (text input, default "PA")
     - ZIP (text input, required)
     - Neighborhood (text input, required -- could be dropdown of existing neighborhoods)
     - Slug (auto-generated from address on create, editable)

     **Section 2: Details**
     - Price (number input, required)
     - Beds (number input, required)
     - Baths (number input, required, allow 0.5 increments)
     - Sqft (number input, required)
     - Lot Sqft (number input)
     - Year Built (number input)
     - Property Type (select: Single Family, Condo, Townhouse, Multi-Family, Land)
     - Status (select: Active, Pending, Sold, Open House, New)

     **Section 3: Description & Features**
     - Description (textarea, required, min 10 chars)
     - Features Interior (tag input -- type and press Enter to add, click X to remove)
     - Features Exterior (same tag input pattern)
     - Features Community (same tag input pattern)

     **Section 4: Media**
     - Images: use ImageUpload component with bucket="property-images" and multiple=true
     - Show existing images as thumbnail grid with X to remove
     - Video URL (text input, optional)
     - Virtual Tour URL (text input, optional)

     **Section 5: Listing Info**
     - Listing Agent (select dropdown of agents, optional)
     - Latitude (number input, optional)
     - Longitude (number input, optional)
     - Open House (text input for date string, optional)
     - Featured (checkbox/toggle)

     **Form behavior:**
     - On create: POST to /api/properties with all fields
     - On edit: PATCH to /api/properties/[id] with changed fields only
     - Submit button: "Create Property" or "Save Changes" (gold styled)
     - Cancel button: calls onCancel
     - Loading state on submit
     - Validation: required fields checked client-side, show inline errors
     - In edit mode: pre-populate all fields from property prop

     **Tag input pattern for features:**
     - Text input with "Press Enter to add" placeholder
     - Tags rendered as pills below input with X button
     - Simple custom component, no external library needed
