# Plan 11-03: Lead Inbox

**Phase:** 11 -- Auth & Agent Portal
**Plan ID:** 11-03
**Status:** PENDING
**Depends On:** 11-02 (dashboard shell with layout, sidebar, header)

## Goal

Build the Lead Inbox dashboard page where agents and admins can view all leads, filter by type/status/agent, click to see detail, assign leads to agents, and update lead status through the pipeline.

## Requirements Addressed

- **AUTH-04**: Lead Inbox -- view, filter by type/status/agent, assign to agents, update status (new -> contacted -> qualified -> closed)

## Context

- Existing `GET /api/leads` route supports `?status=`, `?type=`, `?limit=`, `?offset=` query params and returns `{ data, count, limit, offset }`
- LeadRow type: `{ id, type, first_name, last_name, email, phone, message, property_id, agent_id, status, metadata, ghl_synced, created_at, updated_at }`
- Lead types: "contact", "showing", "seller", "agent-application", "agent-contact"
- Lead statuses: "new", "contacted", "qualified", "closed"
- AgentRow type available for agent dropdown population
- Dashboard layout from 11-02 provides sidebar + header shell

  <action>
  1. Create `src/app/api/leads/[id]/route.ts`:
     - PATCH handler: accepts `{ status?: string, agent_id?: string | null }` in body
     - Validate status is one of: "new", "contacted", "qualified", "closed"
     - If agent_id provided, validate it exists in agents table
     - Update the lead record, set updated_at to now
     - Return updated lead row
     - GET handler: return single lead by ID with joined agent and property data
       - Select: `*, agents!leads_agent_id_fkey(id, full_name, email, phone), properties!leads_property_id_fkey(id, address, slug)`
     - Error handling: 404 if lead not found, 422 for validation errors, 500 for DB errors

  2. Create `src/components/dashboard/lead-table.tsx`:
     - "use client" component
     - Props: `leads: LeadRow[]`, `onRowClick: (lead: LeadRow) => void`, `selectedLeadId?: string`
     - HTML table with columns: Name (first + last), Email, Type, Status, Agent, Date
     - Type column: colored badge (contact=blue, showing=green, seller=purple, agent-application=orange, agent-contact=cyan)
     - Status column: colored badge (new=yellow, contacted=blue, qualified=green, closed=gray)
     - Agent column: show agent name if agent_id exists, otherwise "Unassigned" in muted text
     - Date column: relative time (e.g., "2h ago", "3 days ago") -- use a simple relative time formatter function (no library needed, write a small utility)
     - Selected row highlighted with gold/10 background
     - Row hover: slight bg change
     - Responsive: on mobile, hide email and agent columns, show essential info
     - Sortable by date (default newest first)

  3. Create `src/app/dashboard/leads/page.tsx`:
     - "use client" component (needs interactivity for filters and detail panel)
     - Page heading: "Lead Inbox" with count badge
     - Filter bar at top:
       - Type dropdown: All Types, Contact, Showing, Seller, Agent Application, Agent Contact
       - Status dropdown: All Statuses, New, Contacted, Qualified, Closed
       - Agent dropdown: All Agents, then list of agents from database (fetch agents from /api/agents or inline query)
       - Filters update URL search params and re-fetch leads
     - Fetch leads from `GET /api/leads` with filter params
     - Pass leads to LeadTable component
     - Track selected lead state for detail panel
     - Pagination: "Load more" button or simple prev/next with offset tracking
     - Empty state: "No leads found" with appropriate message
     - Loading state: skeleton rows while fetching
  </action>
