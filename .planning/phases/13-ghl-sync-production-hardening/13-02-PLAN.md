**A. Create src/middleware.ts:**

Implement Next.js middleware with in-memory rate limiting for /api/* routes:

1. Use a Map<string, { count: number; resetAt: number }> for IP tracking (same pattern as the existing rate limiter in leads/route.ts but at middleware level).
2. Config: 10 requests per minute per IP for /api/* routes. This replaces the per-route rate limiting.
3. Extract IP from `x-forwarded-for` header (first IP in chain) or `x-real-ip`, fallback to "unknown".
4. If rate limited, return `NextResponse.json({ error: "Too many requests" }, { status: 429 })`.
5. Add cleanup interval to clear stale entries every 5 minutes.
6. Export `config` with matcher: `["/api/:path*"]` to only run on API routes.
7. For non-API routes, pass through immediately with `NextResponse.next()`.

Note: Do NOT add auth middleware here -- that belongs to the existing auth system from Phase 11. Only add rate limiting.

**B. Create src/components/turnstile.tsx:**

A reusable "use client" component that renders the Cloudflare Turnstile widget:

1. Props: `{ onVerify: (token: string) => void; onExpire?: () => void; className?: string }`.
2. Use the `@marsidev/react-turnstile` package (install it). This is the standard React wrapper for Turnstile.
   - Alternatively, implement manually: load the Turnstile script via next/script, render a div with `cf-turnstile` class, set `data-sitekey` to `process.env.NEXT_PUBLIC_TURNSTILE_SITE_KEY`, and use the `turnstile.render()` callback API.
3. Prefer the manual approach to avoid adding a dependency:
   - Use useRef for the container div.
   - Use useEffect to call `window.turnstile.render(ref.current, { sitekey, callback: onVerify, "expired-callback": onExpire, theme: "dark" })`.
   - Load the script once via `<Script src="https://challenges.cloudflare.com/turnstile/v0/api.js" strategy="afterInteractive" />`.
   - Style: dark theme to match Tauro design. Pass className through for positioning.
4. If NEXT_PUBLIC_TURNSTILE_SITE_KEY is not set, render nothing (graceful degradation for dev).

**A. Update all 5 form components to include Turnstile:**

3. Render `<Turnstile onVerify={setTurnstileToken} onExpire={() => setTurnstileToken("")} className="mt-4" />` just before the submit button.

**B. Update src/app/api/leads/route.ts to validate Turnstile token server-side:**

1. Add `captchaToken: z.string().optional()` to LeadCreateSchema (optional for backward compat, but validated when present).
2. After validation, before DB insert, add Turnstile server-side verification:
   ```ts
   const turnstileSecret = process.env.TURNSTILE_SECRET_KEY;
   if (turnstileSecret) {
     const captchaToken = data.captchaToken;
     if (!captchaToken) {
       return NextResponse.json({ error: "CAPTCHA verification required" }, { status: 400 });
     }
     const verifyRes = await fetch("https://challenges.cloudflare.com/turnstile/v0/siteverify", {
       method: "POST",
       headers: { "Content-Type": "application/x-www-form-urlencoded" },
       body: new URLSearchParams({ secret: turnstileSecret, response: captchaToken, remoteip: ip }),
     });
     const verifyData = await verifyRes.json();
     if (!verifyData.success) {
       return NextResponse.json({ error: "CAPTCHA verification failed" }, { status: 400 });
     }
   }
   ```
3. Remove the per-route rate limiting code from leads/route.ts (lines 9-35, the rateLimitMap and isRateLimited function and the check in POST). This is now handled by middleware.
4. Add basic input sanitization: create a `sanitize(input: string): string` helper at the top of the file that strips HTML tags using a simple regex (`input.replace(/<[^>]*>/g, "").trim()`). Apply it to firstName, lastName, email, phone, message before DB insert. This prevents stored XSS.
5. Do NOT strip captchaToken from the data before DB insert -- exclude it when building the insert object (don't pass it to Supabase).
