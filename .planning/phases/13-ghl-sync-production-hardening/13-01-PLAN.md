<task type="auto">
  <name>Task 2: Create inbound GHL webhook route and refactor leads route</name>

**A. Create src/app/api/webhooks/ghl/route.ts:**

POST handler that:
1. Reads raw body using `await request.text()` (needed for signature verification).
2. Extracts signature from `x-ghl-signature` header (or `x-webhook-signature` -- check GHL docs, support both).
3. Calls `verifyGhlSignature(rawBody, signature)` from src/lib/ghl.ts. Return 401 if invalid.
4. Parse the JSON body. Expected GHL webhook payload shape:
   ```
   {
     type: "ContactStatusChanged" | "ContactTagUpdated" | string,
     contact: {
       id: string,
       email: string,
       phone: string,
       tags: string[],
       customFields: Record<string, string>
     },
     status?: string
   }
   ```
5. For "ContactStatusChanged" events: Look up the lead in Supabase by email (from contact.email). Map GHL status to Tauro lead status:
   - "open" or "new" -> "new"
   - "contacted" -> "contacted"
   - "qualified" -> "qualified"
   - "won" or "closed-won" -> "closed"
   - "lost" or "closed-lost" -> "closed"
   Update the lead's `status` and set `ghl_synced: true`, `updated_at: new Date().toISOString()`.
6. Return 200 with `{ received: true }`.
7. If GHL_WEBHOOK_SECRET is not set, log a warning but still process (graceful degradation for development).

5. Keep all other logic (rate limiting, validation, DB insert, GET handler) unchanged.
