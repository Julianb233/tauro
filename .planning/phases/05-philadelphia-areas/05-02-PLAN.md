Create `src/app/(site)/neighborhoods/[slug]/page.tsx` as a SERVER component.

Follow the EXACT pattern from `src/app/properties/[slug]/page.tsx` for Next.js 16 App Router conventions:
- `params` is `Promise<{ slug: string }>` (must await)
- `generateStaticParams` returns array of `{ slug: string }`
- `generateMetadata` accepts `{ params: Promise<{ slug: string }> }`

**generateStaticParams:**
```typescript
import { neighborhoods, getNeighborhoodBySlug } from "@/data/neighborhoods";
import { properties } from "@/data/properties";

export function generateStaticParams() {
  return neighborhoods.map((n) => ({ slug: n.slug }));
}
```

**generateMetadata:**
```typescript
export async function generateMetadata({ params }: { params: Promise<{ slug: string }> }) {
  const { slug } = await params;
  const neighborhood = getNeighborhoodBySlug(slug);
  if (!neighborhood) return { title: "Neighborhood Not Found | Tauro" };
  return {
    title: `${neighborhood.name} Homes for Sale | Tauro`,
    description: `Explore homes for sale in ${neighborhood.name}, Philadelphia. ${neighborhood.tagline} Browse listings, local insights, and market data with Tauro Real Estate.`,
  };
}
```

**Page sections (top to bottom):**

1. **Hero**: Full-width image with dark gradient overlay. Neighborhood name as h1 (Playfair via `font-heading`), tagline below. Breadcrumb: Home > Neighborhoods > {Name}.

2. **About section**: The neighborhood description text. Use `prose` or manual styling with `text-muted-foreground` body text and `text-foreground` headings.

3. **Selling Points**: Grid (2 col on md) of selling points with checkmark or MapPin icons. Gold icon color (`text-gold`).

4. **Lifestyle section**: 2x2 grid showing the 4 lifestyle categories (vibe, dining, transit, parks) each with an icon and description. Use icons from lucide-react: Coffee for dining, Train for transit, Trees for parks, Sparkles for vibe.

5. **Market Stats**: Horizontal row of 4 stat cards (median price, avg $/sqft, avg days on market, inventory level). Dark cards (`bg-card border border-border`) with gold value text.

6. **Properties section**: Filter `properties` array where `property.city` matches `neighborhood.propertyFilter` (case-insensitive compare). Display matching properties using the existing `PropertyCard` component in a responsive grid. If no matches (likely with current placeholder data), show a tasteful empty state: "New listings in {neighborhood.name} coming soon. Contact us to be the first to know." with a link to /contact.

7. **Map placeholder**: A dark card with `bg-card` showing "Map of {neighborhood.name}" text and a note that interactive map is coming. Style it as a 16:9 aspect ratio placeholder with a MapPin icon centered.

8. **CTA section**: "Interested in {neighborhood.name}?" with buttons to /contact and /properties.

Import `PropertyCard` from `@/components/PropertyCard`. Use lucide-react icons. Keep the page as a server component (no "use client"). All styling with Tailwind, dark theme consistent.
