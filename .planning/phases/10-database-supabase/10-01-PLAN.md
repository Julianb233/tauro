  <name>Task 1: Install Supabase packages and create typed clients</name>
  <files>
    package.json
    src/lib/supabase/client.ts
    src/lib/supabase/server.ts
    .env.example
    .env.local

  <action>
    1. Install packages:
       ```
       npm install @supabase/supabase-js @supabase/ssr
       ```

    2. Create `src/lib/supabase/client.ts` — browser client:
       - Import `createBrowserClient` from `@supabase/ssr`
       - Import `Database` type from `@/types/database`
       - Export a function `createClient()` that returns `createBrowserClient<Database>(url, anonKey)`
       - Read url and anonKey from `process.env.NEXT_PUBLIC_SUPABASE_URL!` and `process.env.NEXT_PUBLIC_SUPABASE_ANON_KEY!`

    3. Create `src/lib/supabase/server.ts` — server client for Next.js App Router:
       - Import `createServerClient` from `@supabase/ssr`
       - Import `cookies` from `next/headers`
       - Import `Database` type from `@/types/database`
       - Export an async function `createClient()` that:
         a. Awaits `cookies()` to get the cookie store
         b. Returns `createServerClient<Database>(url, anonKey, { cookies: { getAll() { return cookieStore.getAll() }, setAll(cookiesToSet) { cookiesToSet.forEach(({ name, value, options }) => cookieStore.set(name, value, options)) } } })`
       - This pattern follows the official Supabase + Next.js App Router docs

    4. Create `.env.example`:
       ```
       NEXT_PUBLIC_SUPABASE_URL=your-project-url
       NEXT_PUBLIC_SUPABASE_ANON_KEY=your-anon-key
       GHL_WEBHOOK_URL=your-ghl-webhook-url
       ```

    5. Create `.env.local` with the same keys but empty values (user will fill in):
       ```
       NEXT_PUBLIC_SUPABASE_URL=
       NEXT_PUBLIC_SUPABASE_ANON_KEY=
       ```
       If `.env.local` already exists, append the Supabase vars without overwriting existing content.
  </action>
  <verify>
    - `npm ls @supabase/supabase-js @supabase/ssr` shows both installed
    - `npx tsc --noEmit src/lib/supabase/client.ts` compiles (once types exist from Task 2)
    - `.env.example` exists with all required vars
  </verify>
  <done>
    Supabase packages installed. Browser and server clients created with proper Database generic typing. Env template documents all required variables.

<task type="auto">
  <name>Task 2: Create database types and SQL migration</name>
  <files>
    src/types/database.ts
    supabase/migrations/001_initial_schema.sql
  </files>
  <action>
    1. Create `src/types/database.ts` with the full Supabase Database type. Map existing TS interfaces to database tables:

    **properties table** (from Property interface):
    - id: uuid (PK, default gen_random_uuid())
    - slug: text (unique, not null)
    - address: text (not null)
    - city: text (not null, default 'Philadelphia')
    - state: text (not null, default 'PA')
    - zip: text (not null)
    - neighborhood: text (not null)
    - price: integer (not null)
    - beds: smallint (not null)
    - baths: numeric(3,1) (not null) — allows half baths like 2.5
    - sqft: integer (not null)
    - lot_sqft: integer (not null, default 0)
    - year_built: smallint
    - status: text (not null, default 'Active') — values: Active, Pending, Sold, Open House, New
    - property_type: text (not null) — values: Single Family, Condo, Townhouse, Multi-Family, Land
    - images: text[] (not null, default '{}')
    - description: text (not null)
    - features_interior: text[] (default '{}')
    - features_exterior: text[] (default '{}')
    - features_community: text[] (default '{}')
    - listing_agent_id: uuid (FK to agents.id, nullable)
    - lat: double precision
    - lng: double precision
    - open_house: text (nullable)
    - video_url: text (nullable)
    - virtual_tour_url: text (nullable)
    - featured: boolean (default false)
    - created_at: timestamptz (default now())
    - updated_at: timestamptz (default now())

    **agents table** (from Agent interface):
    - id: uuid (PK, default gen_random_uuid())
    - slug: text (unique, not null)
    - first_name: text (not null)
    - last_name: text (not null)
    - full_name: text (generated always as first_name || ' ' || last_name, stored)
    - title: text (not null)
    - email: text (unique, not null)
    - phone: text (not null)
    - photo: text
    - bio: text
    - short_bio: text
    - specialties: text[] (default '{}')
    - neighborhoods: text[] (default '{}')
    - stats: jsonb (default '{}') — { propertiesSold, totalVolume, avgDaysOnMarket, yearsExperience }
    - awards: jsonb (default '[]') — array of { title, year, issuer }
    - video_intro_url: text (nullable)
    - social: jsonb (default '{}') — { instagram?, linkedin?, facebook? }
    - languages: text[] (default '{}')
    - license_number: text
    - created_at: timestamptz (default now())
    - updated_at: timestamptz (default now())

    **neighborhoods table** (from Neighborhood interface):
    - id: uuid (PK, default gen_random_uuid())
    - slug: text (unique, not null)
    - name: text (not null)
    - tagline: text
    - description: text
    - selling_points: text[] (default '{}')
    - lifestyle: jsonb (default '{}') — { vibe, dining, transit, parks }
    - stats: jsonb (default '{}') — { medianPrice, avgPricePerSqft, avgDaysOnMarket, inventoryLevel }
    - image: text
    - card_image: text
    - map_center: jsonb — { lat, lng }
    - property_filter: text
    - featured: boolean (default false)
    - created_at: timestamptz (default now())
    - updated_at: timestamptz (default now())

    **leads table** (from LeadPayload interface):
    - id: uuid (PK, default gen_random_uuid())
    - type: text (not null) — values: contact, showing, seller, agent-application, agent-contact
    - first_name: text (not null)
    - last_name: text (not null)
    - email: text (not null)
    - phone: text (not null)
    - message: text
    - property_id: uuid (FK to properties.id, nullable)
    - agent_id: uuid (FK to agents.id, nullable)
    - status: text (not null, default 'new') — values: new, contacted, qualified, closed
    - metadata: jsonb (default '{}') — stores all extra fields (propertyAddress, preferredDate, etc.)
    - ghl_synced: boolean (default false)
    - created_at: timestamptz (default now())
    - updated_at: timestamptz (default now())

    **testimonials table**:
    - id: uuid (PK, default gen_random_uuid())
    - quote: text (not null)
    - name: text (not null)
    - role: text
    - rating: smallint (default 5)
    - sort_order: smallint (default 0)
    - created_at: timestamptz (default now())

    **faq table**:
    - id: uuid (PK, default gen_random_uuid())
    - question: text (not null)
    - answer: text (not null)
    - category: text (not null, default 'general') — values: buyer, seller, general
    - sort_order: smallint (default 0)
    - created_at: timestamptz (default now())

    The `src/types/database.ts` file must follow the Supabase generated types pattern:
    ```typescript
    export type Database = {
      public: {
        Tables: {
          properties: {
            Row: { ... all columns with TS types ... }
            Insert: { ... required + optional columns ... }
            Update: { ... all optional ... }
          }
          // ... same for each table
        }
      }
    }
    ```

    Also export convenience aliases:
    ```typescript
    export type PropertyRow = Database['public']['Tables']['properties']['Row']
    export type AgentRow = Database['public']['Tables']['agents']['Row']
    export type NeighborhoodRow = Database['public']['Tables']['neighborhoods']['Row']
    export type LeadRow = Database['public']['Tables']['leads']['Row']
    export type TestimonialRow = Database['public']['Tables']['testimonials']['Row']
    export type FaqRow = Database['public']['Tables']['faq']['Row']
    ```

    2. Create `supabase/migrations/001_initial_schema.sql`:
    - CREATE TABLE statements for all 6 tables matching the types above
    - Add indexes: properties(slug), properties(neighborhood), properties(status), agents(slug), agents(email), neighborhoods(slug), leads(status), leads(type), leads(created_at)
    - Add FK constraints: properties.listing_agent_id -> agents.id (SET NULL on delete), leads.property_id -> properties.id (SET NULL), leads.agent_id -> agents.id (SET NULL)
    - Add an `updated_at` trigger function and apply it to properties, agents, neighborhoods, leads tables
    - Use snake_case for all column names (NOT camelCase)
    - Add comments on tables describing their purpose

    IMPORTANT: Do NOT use Prisma. This project uses raw Supabase client with SQL migrations.
    IMPORTANT: The `full_name` column on agents should be a GENERATED column: `full_name text GENERATED ALWAYS AS (first_name || ' ' || last_name) STORED`
  </action>

  </verify>
