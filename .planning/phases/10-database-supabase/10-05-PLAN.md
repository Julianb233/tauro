    The key challenge: database rows use snake_case (property_type, listing_agent_id) while current components expect camelCase (propertyType, listingAgent). Two approaches:

    **Approach: Create a mapping utility.** Add to `src/lib/supabase/queries.ts` or a new `src/lib/supabase/mappers.ts`:

    ```typescript
    import type { PropertyRow, AgentRow } from '@/types/database';
    import type { Property } from '@/data/properties';
    import type { Agent } from '@/data/agents';

    export function mapPropertyRowToProperty(row: PropertyRow & { agents?: any }): Property {
      return {
        id: row.id,
        slug: row.slug,
        address: row.address,
        city: row.city,
        state: row.state,
        zip: row.zip,
        neighborhood: row.neighborhood,
        price: row.price,
        beds: row.beds,
        baths: row.baths,
        sqft: row.sqft,
        lotSqft: row.lot_sqft,
        yearBuilt: row.year_built || 0,
        status: row.status as Property['status'],
        propertyType: row.property_type as Property['propertyType'],
        images: row.images,
        description: row.description,
        features: {
          interior: row.features_interior || [],
          exterior: row.features_exterior || [],
          community: row.features_community || [],
        },
        agent: row.agents ? {
          name: row.agents.full_name,
          phone: row.agents.phone,
          email: row.agents.email,
          photo: row.agents.photo || '',
        } : { name: '', phone: '', email: '', photo: '' },
        lat: row.lat || 0,
        lng: row.lng || 0,
        openHouse: row.open_house || undefined,
        videoUrl: row.video_url || undefined,
        virtualTourUrl: row.virtual_tour_url || undefined,
      };
    }

    export function mapAgentRowToAgent(row: AgentRow): Agent {
      return {
        id: row.id,
        slug: row.slug,
        firstName: row.first_name,
        lastName: row.last_name,
        fullName: row.full_name || `${row.first_name} ${row.last_name}`,
        title: row.title,
        email: row.email,
        phone: row.phone,
        photo: row.photo || '',
        bio: row.bio || '',
        shortBio: row.short_bio || '',
        specialties: row.specialties || [],
        neighborhoods: row.neighborhoods || [],
        stats: row.stats as Agent['stats'],
        awards: (row.awards as any[]) || [],
        videoIntroUrl: row.video_intro_url || null,
        activeListingIds: [],  // Will be populated from properties query
        soldListingIds: [],
        social: row.social as Agent['social'],
        languages: row.languages || [],
        licenseNumber: row.license_number || '',
      };
    }
    ```

    This approach preserves ALL existing component interfaces — PropertyCard, AgentCard, etc. keep using the same types. Only the data source changes.

    **Now migrate each page:**

    1. **src/app/properties/page.tsx** (server component):
       - Remove: `import { properties } from "@/data/properties"`
       - Add: `import { getProperties } from "@/lib/supabase/queries"` and `import { mapPropertyRowToProperty } from "@/lib/supabase/mappers"`
       - In the component body, call `const { data } = await getProperties()` and map: `const properties = data.map(mapPropertyRowToProperty)`
       - The rest of the component stays the same (it already uses the Property type)
       - NOTE: If this page currently passes properties to a client component, keep that pattern but fetch from DB instead of static import

    2. **src/app/properties/[slug]/page.tsx** (server component):
       - Remove static import
       - Call `getPropertyBySlug(slug)` and map with `mapPropertyRowToProperty`
       - If not found, call `notFound()`
       - For `generateStaticParams`: query all property slugs from DB
         ```typescript
         export async function generateStaticParams() {
           const { data } = await getProperties({ limit: 1000 });
           return data.map(p => ({ slug: p.slug }));
         }
         ```
       - Add revalidation: `export const revalidate = 3600` (1 hour)

    3. **src/app/properties/[slug]/PropertyDetailClient.tsx** (client component):
       - This receives Property as a prop from the server component — NO changes needed if the mapper works correctly
       - Only change if it directly imports from `@/data/properties` for the type — keep the type import but remove data import

    4. **src/app/(site)/agents/page.tsx** (server component):
       - Remove: `import { agents } from "@/data/agents"`
       - Add: `import { getAgents } from "@/lib/supabase/queries"` and mapper
       - Fetch and map agents
       - Add `export const revalidate = 3600`

    5. **src/app/(site)/agents/[slug]/page.tsx** (server component):
       - Fetch agent by slug from DB
       - Also fetch agent's properties: `getProperties({ listing_agent_id: agent.id })`
       - Map both
       - `generateStaticParams` from DB
       - `revalidate = 3600`

    6. **src/app/(site)/agents/[slug]/AgentProfileClient.tsx** (client component):
       - Remove data imports, keep type imports
       - It receives agent data as props from server component

    7. **Component type imports**: PropertyCard.tsx, PropertyMap.tsx, AgentCard.tsx, PropertyFilters.tsx currently import TYPES from @/data/*.ts. Keep these type imports — they still work since we're mapping DB rows to the same types. Do NOT change the component interfaces.

    IMPORTANT: Do NOT change the shapes of PropertyCard, AgentCard, or any shared component props. The mapper functions handle the translation from database rows to the existing TypeScript interfaces. This is critical for avoiding a cascade of changes.

    IMPORTANT: For pages with `generateStaticParams`, add `export const revalidate = 3600` for ISR (Incremental Static Regeneration) so pages rebuild hourly when data changes.
