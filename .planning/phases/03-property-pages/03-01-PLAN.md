<task type="auto">
  <name>Task 2: Create MapView and ImageGallery components + marker CSS</name>
  <files>src/components/MapView.tsx, src/components/ImageGallery.tsx, src/app/globals.css</files>
  <action>
    **Create `src/components/MapView.tsx`** — a "use client" Mapbox GL JS component:
    - Import mapboxgl and its CSS (`mapbox-gl/dist/mapbox-gl.css`)
    - Set `mapboxgl.accessToken` from `process.env.NEXT_PUBLIC_MAPBOX_TOKEN!`
    - Props: `properties: Property[]`, `onMarkerClick?: (property: Property) => void`, `center?: [number, number]`, `zoom?: number`, `singleMarker?: boolean` (for detail page use with one property)
    - Use `useRef` for map container and map instance. Guard init with `if (map.current) return`
    - Style: `mapbox://styles/mapbox/dark-v11`
    - Default center: `[-117.16, 32.78]` (San Diego area — matches existing property data)
    - Default zoom: 10
    - Add `NavigationControl` at top-right
    - For each property, create a custom DOM marker element with class `mapbox-price-marker` showing `formatPrice(property.price)`. Use `mapboxgl.Marker({ element: el })` positioned at `[property.lng, property.lat]`
    - Add click listener on each marker element to call `onMarkerClick`
    - After adding all markers, call `map.current.fitBounds(bounds, { padding: 60, maxZoom: 14 })` to fit all properties in view (skip fitBounds if singleMarker is true — use center/zoom instead)
    - Cleanup: `map.current?.remove(); map.current = null` in useEffect cleanup
    - Render: `<div ref={mapContainer} className="h-full w-full rounded-xl" />`
    - Handle case where NEXT_PUBLIC_MAPBOX_TOKEN is empty/missing: show a styled fallback div with "Map unavailable — configure Mapbox token" message (so the app doesn't crash without a token)

    **Create `src/components/ImageGallery.tsx`** — a "use client" lightbox gallery:
    - Import Lightbox from `yet-another-react-lightbox` and plugins: Counter, Fullscreen, Thumbnails, Zoom
    - Import CSS: `yet-another-react-lightbox/styles.css`, `yet-another-react-lightbox/plugins/counter/counter.css`, `yet-another-react-lightbox/plugins/thumbnails/thumbnails.css`
    - Props: `images: string[]`, `address: string`
    - State: `open: boolean`, `index: number`
    - Render a thumbnail grid: first image spans full width (aspect-[16/9], max-h-[500px]), below it 3-4 smaller thumbnails in a row. If >4 images total, last thumbnail shows "+N more" overlay. Clicking any thumbnail opens the lightbox at that index.
    - Lightbox config: `plugins={[Counter, Fullscreen, Thumbnails, Zoom]}`, `styles={{ container: { backgroundColor: "rgba(0, 0, 0, 0.95)" } }}`
    - Slides: `images.map(src => ({ src }))`
    - Use `/* eslint-disable-next-line @next/next/no-img-element */` before img tags (matching existing codebase pattern — project uses Unsplash URLs, not local images)

    **Append to `src/app/globals.css`** — add Mapbox price marker styles at the end of the file:
    ```css
    /* Mapbox property price markers */
    .mapbox-price-marker {
      background: var(--color-midnight);
      color: var(--color-gold);
      padding: 4px 8px;
      border-radius: 9999px;
      font-size: 12px;
      font-weight: 700;
      font-family: var(--font-sans);
      cursor: pointer;
      white-space: nowrap;
      box-shadow: 0 2px 8px rgba(0, 0, 0, 0.3);
      transition: background 0.2s, color 0.2s;
    }
    .mapbox-price-marker:hover {
      background: var(--color-gold);
      color: var(--color-near-black);
    }
    ```

    Do NOT use next/dynamic or ssr:false — the "use client" + useEffect pattern is the correct App Router approach for DOM-dependent libraries.
  </action>
  <verify>
    Run `npx tsc --noEmit` to verify TypeScript compilation passes.

  </verify>
