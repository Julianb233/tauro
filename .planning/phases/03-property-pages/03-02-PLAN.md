  <files>src/app/properties/page.tsx, src/components/PropertyFilters.tsx</files>
  <action>
    **Refactor `src/components/PropertyFilters.tsx`:**
    - Remove the internal `FilterState` type export and `defaultFilters` export (these will be derived from URL params in the page)
    - Change props to: `filters: Record<string, string>`, `onChange: (key: string, value: string) => void`, `onClear: () => void`
    - The component becomes a pure presentation layer — it renders select elements and calls `onChange(key, value)` on change
    - Keep the exact same UI (mobile toggle, filter bar, sort, clear all button)
    - Keep the `activeCount` calculation to show filter badge count
    - Do NOT add useSearchParams here — the parent page owns the URL state

    **Refactor `src/app/properties/page.tsx`:**
    - Wrap the main page content in a `<Suspense>` boundary (required by Next.js when using useSearchParams)
    - Create an inner component (e.g., `PropertiesContent`) that uses `useSearchParams()` and `useRouter()`
    - Read filter values from search params: `priceMin`, `priceMax`, `beds`, `baths`, `type` (propertyType), `status`, `sort` (default "price-desc")
    - Create `updateFilter(key: string, value: string)` function that builds new URLSearchParams, removes empty/default values, calls `router.replace(\`/properties?\${params}\`, { scroll: false })`
    - Create `clearFilters()` that calls `router.replace("/properties", { scroll: false })`
    - Pass these to PropertyFilters: `filters={currentFilters}`, `onChange={updateFilter}`, `onClear={clearFilters}`
    - The `useMemo` filtering logic stays the same but reads from the URL-derived filter object
    - Import `Suspense` from react: `import { Suspense } from "react"`
    - The default export renders `<Suspense fallback={<div>Loading...</div>}><PropertiesContent /></Suspense>`

    Important: useSearchParams requires a Suspense boundary in App Router. Without it, the build will fail or show a warning.
  </action>
