    Place these after the Baths select and before the Type select. The filter bar grid on mobile (grid-cols-2) wraps naturally. On desktop (lg:flex) they sit in a row — selects are compact enough.

    **Refactor `src/app/properties/page.tsx`:**
    - Add imports: `import { Suspense } from "react"` and `import { useSearchParams, useRouter } from "next/navigation"`
    - Remove import of `MapPlaceholder`
    - Add import: `import PropertyMap from "@/components/PropertyMap"`
    - Keep import of `{ FilterState, defaultFilters }` from PropertyFilters

    - Split into outer wrapper + inner component:
      - The default export becomes a Suspense wrapper:
        ```tsx
        export default function PropertiesPage() {
          return (
            <Suspense fallback={<div className="min-h-screen pt-16" />}>
              <PropertiesContent />
            </Suspense>
          );
        }
        ```
      - Move all current logic into `function PropertiesContent()` which uses:
        - `const searchParams = useSearchParams()`
        - `const router = useRouter()`
        - `const [view, setView] = useState<ViewMode>("grid")`

    - Read filter values from search params into a FilterState object using `useMemo`:
      ```tsx
      const filters: FilterState = useMemo(() => ({
        priceMin: searchParams.get("priceMin") || "",
        priceMax: searchParams.get("priceMax") || "",
        beds: searchParams.get("beds") || "",
        baths: searchParams.get("baths") || "",
        sqftMin: searchParams.get("sqftMin") || "",
        sqftMax: searchParams.get("sqftMax") || "",
        area: searchParams.get("area") || "",
        propertyType: searchParams.get("type") || "",
        status: searchParams.get("status") || "",
        sort: searchParams.get("sort") || "price-desc",
      }), [searchParams]);
      ```

    - Create `updateFilter` callback:
      ```tsx
      const updateFilter = useCallback((key: keyof FilterState, value: string) => {
        const params = new URLSearchParams(searchParams.toString());
        const paramKey = key === "propertyType" ? "type" : key;
        if (value && !(key === "sort" && value === "price-desc")) {
          params.set(paramKey, value);
        } else {
          params.delete(paramKey);
        }
        const qs = params.toString();
        router.replace(`/properties${qs ? `?${qs}` : ""}`, { scroll: false });
      }, [searchParams, router]);
      ```

    - Create `clearFilters` callback:
      ```tsx
      const clearFilters = useCallback(() => {
        router.replace("/properties", { scroll: false });
      }, [router]);
      ```

    - **Add filtering logic for the new filters in the `useMemo` filtering block:**
      - `if (filters.sqftMin) { result = result.filter((p) => p.sqft >= Number(filters.sqftMin)); }`
      - `if (filters.sqftMax) { result = result.filter((p) => p.sqft <= Number(filters.sqftMax)); }`
      - `if (filters.area) { result = result.filter((p) => p.city === filters.area); }`

    - The existing price/beds/baths/type/status filtering logic stays the same but now reads from the URL-derived `filters` object
    - Pass to PropertyFilters: `<PropertyFilters filters={filters} onChange={updateFilter} onClear={clearFilters} />`
    - Remove the `useState<FilterState>` -- it is fully replaced by URL params
    - Update the "Clear all filters" button in the empty state to call `clearFilters()` instead of `setFilters(defaultFilters)`

    Important: useSearchParams requires a Suspense boundary in App Router. Without it, the build will fail.
