<task type="auto">
  <name>Task 1: Install dependencies and create email service</name>
  <files>package.json, src/lib/email.ts</files>
  <action>
    1. Install resend and @react-email/components:
       `npm install resend @react-email/components`

    2. Create `src/lib/email.ts` with:
       - Import Resend from 'resend'
       - Instantiate: `const resend = new Resend(process.env.RESEND_API_KEY)`
       - Constants: `EMAIL_FROM = process.env.EMAIL_FROM || 'Tauro Realty <noreply@taurorealty.com>'` and `ADMIN_EMAIL = process.env.ADMIN_EMAIL || 'admin@lylrealty.com'`
       - Export async function `sendLeadConfirmation(to: string, data: { firstName: string; type: string; message?: string })` -- renders LeadConfirmation template, sends via resend.emails.send. Returns `{ success: boolean; error?: string }`.
       - Export async function `sendAgentNotification(to: string, data: { leadName: string; leadEmail: string; leadPhone: string; leadType: string; message?: string; propertyAddress?: string })` -- renders AgentNotification template, sends to agent email. Returns same shape.
       - Export async function `sendApplicationConfirmation(to: string, data: { firstName: string; licenseNumber?: string })` -- renders ApplicationConfirmation template. Returns same shape.
       - All functions wrap in try/catch, log errors with console.error, return `{ success: false, error: message }` on failure. Never throw -- callers should not fail if email fails.

    Important: Use `render()` from `@react-email/components` to convert React Email components to HTML string, then pass as `html` to `resend.emails.send()`. Do NOT use the `react` property of resend.emails.send (it requires specific React Email version alignment that causes issues).
  </action>
  <verify>
    - `npx tsc --noEmit` passes (no type errors)
    - `src/lib/email.ts` exports all three send functions
    - Resend appears in package.json dependencies
  </verify>
  <done>Email service module exists with three typed send functions that gracefully handle failures.</done>
